--- layout: control permalink: /CCE-26778-1/ name: CCE-26778-1 title: Add nodev Option to /dev/shm ident: CCE-26778-1 severity: low rule_id: mount_option_dev_shm_nodev ---

Failing to add nodev option to /dev/shm is of low severity.

Rationale

The only legitimate location for device files is the /dev directory located on the root partition. The only exception to this is chroot jails.

via FISMA SSG. View all 11 Restrict Partition Mount Options rules.

Technical Description

The nodev mount option can be used to prevent creation of device files in /dev/shm. Legitimate character and block devices should not exist within temporary directories like /dev/shm. Add the nodev option to the fourth column of /etc/fstab for the line which controls mounting of /dev/shm.
