--- layout: control permalink: /CCE-27038-9/ name: CCE-27038-9 title: Prevent Log In to Accounts With Empty Password ident: CCE-27038-9 severity: high ---

If an account has an empty password, anyone could log in and run commands with the privileges of that account. Accounts with empty passwords should never be used in operational environments.

Failing to prevent log in to accounts with empty password is of high severity.

View all 4 Verify Proper Storage and Existence of Password Hashes rules.

Technical Description

If an account is configured for password authentication but does not have an assigned password, it may be possible to log into the account without authentication. Remove any instances of the nullok option in /etc/pam.d/system-auth to prevent logins with empty passwords.
