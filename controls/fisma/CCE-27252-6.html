Failing to disable control group rules engine (cgred) is of low severity.

Rationale

Unless control groups are used to manage system resources, running the cgred service service is not necessary.

via FISMA SSG. View all 24 Base Services rules.

Technical Description

The cgred service moves tasks into control groups according to parameters set in the /etc/cgrules.conf configuration file. The cgred service can be disabled with the following command: # chkconfig cgred off
