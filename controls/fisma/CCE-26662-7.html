Failing to ensure auditd collects system administrator actions is of low severity.

Rationale

The actions taken by system administrators should be audited to keep a record of what was executed on the system, as well as, for accountability purposes.

via FISMA SSG. View all 14 Configure auditd Rules for Comprehensive Auditing rules.

Technical Description

At a minimum the audit system should collect administrator actions for all users and root. Add the following to /etc/audit/audit.rules: 

-w /etc/sudoers -p wa -k actions
