--- layout: control permalink: /CCE-27110-6/ name: CCE-27110-6 title: Set Lockout Time For Failed Password Attempts ident: CCE-27110-6 severity: medium ---

Locking out user accounts after a number of incorrect attempts prevents direct password guessing attacks. Ensuring that an administrator is involved in unlocking locked accounts draws appropriate attention to such situations.

Failing to set lockout time for failed password attempts is of medium severity.

View all 4 Set Lockouts for Failed Password Attempts rules.

Technical Description

To configure the system to lock out accounts after a number of incorrect login attempts and require an administrator to unlock the account using pam_faillock.so:

Add the following lines immediately below the pam_env.so statement in /etc/pam.d/system-auth: 

auth [default=die] pam_faillock.so authfail deny=3 unlock_time=604800 fail_interval=900
auth required pam_faillock.so authsucc deny=3 unlock_time=604800 fail_interval=900
