--- layout: control permalink: /CCE-26282-4/ name: CCE-26282-4 title: Set SSH Client Alive Count ident: CCE-26282-4 severity: low rule_id: sshd_set_keepalive ---

Failing to set ssh client alive count is of low severity.

Rationale

This ensures a user login will be terminated as soon as the ClientAliveCountMax is reached.

via FISMA SSG. View all 11 Configure OpenSSH Server if Necessary rules.

Technical Description

To ensure the SSH idle timeout occurs precisely when the ClientAliveCountMax is set, edit /etc/ssh/sshd_config as follows: 

ClientAliveCountMax 0
