--- layout: control permalink: /CCE-26915-9/ name: CCE-26915-9 title: Enable Kernel Parameter to Use Reverse Path Filtering by Default ident: CCE-26915-9 severity: medium ---

Enabling reverse path filtering drops packets with source addresses that should not have been able to be received on the interface they were received on. It should not be used on systems which are routers for complicated networks, but is helpful for end hosts and routers serving small networks.

Failing to enable kernel parameter to use reverse path filtering by default is of medium severity.

View all 12 Network Related Kernel Runtime Parameters for Hosts and Routers rules.

Technical Description

To set the runtime status of the net.ipv4.conf.default.rp_filter kernel parameter, run the following command: # sysctl -w net.ipv4.conf.default.rp_filter=1 If this is not the system's default value, add the following line to /etc/sysctl.conf: net.ipv4.conf.default.rp_filter = 1
