--- layout: control permalink: /CCE-26812-8/ name: CCE-26812-8 title: Ensure Log Files Are Owned By Appropriate User ident: CCE-26812-8 severity: medium ---

The log files generated by rsyslog contain valuable information regarding system configuration, user authentication, and other such information. Log files should be protected from unauthorized access.

Failing to ensure log files are owned by appropriate user is of medium severity.

View all 3 Ensure Proper Configuration of Log Files rules.

Technical Description

The owner of all log files written by rsyslog should be root. These log files are determined by the second part of each Rule line in /etc/rsyslog.conf and typically all appear in /var/log. For each log file LOGFILE referenced in /etc/rsyslog.conf, run the following command to inspect the file's owner: 

$ ls -l LOGFILE
If the owner is not root, run the following command to correct this: 
# chown root LOGFILE
