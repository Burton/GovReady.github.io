--- layout: control permalink: /CCE-26803-7/ name: CCE-26803-7 title: Ensure rsyslog Does Not Accept Remote Messages Unless Acting As Log Server ident: CCE-26803-7 severity: low ---

Any process which receives messages from the network incurs some risk of receiving malicious messages. This risk can be eliminated for rsyslog by configuring it not to listen on the network.

Failing to ensure rsyslog does not accept remote messages unless acting as log server is of low severity.

View all 3 Configure rsyslogd to Accept Remote Messages If Acting as a Log Server rules.

Technical Description

The rsyslog daemon should not accept remote messages unless the system acts as a log server. To ensure that it is not listening on the network, ensure the following lines are not found in /etc/rsyslog.conf: 

$ModLoad imtcp.so
$InputTCPServerRun port
$ModLoad imudp.so
$InputUDPServerRun port
$ModLoad imrelp.so
$InputRELPServerRun port
