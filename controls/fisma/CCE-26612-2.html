Failing to make the auditd configuration immutable is of low severity.

Rationale

Making the audit configuration immutable prevents accidental as well as malicious modification of the audit rules, although it may be problematic if legitimate changes are needed during system operation

via FISMA SSG. View all 14 Configure auditd Rules for Comprehensive Auditing rules.

Technical Description

Add the following to /etc/audit/audit.rules in order to make the configuration immutable: 

-e 2
With this setting, a reboot will be required to change any audit rules.
